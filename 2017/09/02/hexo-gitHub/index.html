搭建hexo+gitHub中出现的一些错误

Sep 2, 2017建站前端阅读次数 : -

守着善良，伴我成长。
人生本不易，且行且珍惜

偶然间看到一些大神的网站，很美很有个人特色，而且又是他们自己的。就在想，如果自己也有一个那该多好。。。于是，开始吧。

下面简要说一下大致的步骤，不详细说了，详细的网上很多教程，随便搜一下就行，懒得搜那我随便推荐一个吧

环境配置

首先你的电脑得有node,git,github账号
我的node版本是6.11.1
git版本是2.9.3
node.js和Git Bash,注意，git bash我给的不是官网，你最好自己去官网下载安装。

GitHub的话,注册一下，注册之后需要新建仓库，也就是你保存代码的地方

剪完之后，单击进你的项目，比如我的hexiuhui.github.io

（注意：这个组合是固定的，必须是你的github用户名+github.io）进去你自己的项目之后，看到Code Issues点击后面的Settings，然后往下滑，看到GitHub Pages，它的Source是None，选master branch,在随便选一个主题，保存之后，你就可以访问你的GitHub Pages了。在浏览器地址栏输入hexiuhui.github.io，换成你自己的试试。

然后github还需要做的就是添加SSH，为了让你在上传本地代码到github的时候不需要每次上传都输入密码。这个其实很方便的。打开Git Bash,输入下面代码

安利一个很好的教程：window下配置SSH连接GitHub、GitHub配置ssh key

创建一个文件，比如blog，进入blog，然后右键打开你的Git Bash,

这个修改完，在加上之前的hexiuhui.github.io,在加上ssh就可以使用hexo d来更新远程博客了。

写博客

写文章的话，是在这个路径blog\source_posts，在里面新建.md文件就可以了(可以手动新建txt文本在修改后缀名，也可以使用hexo new 'name')，这一个文件就是你的一篇文章。如你所见，你hexo s之后看到的hello World的页面就是这里相应的文件。md语法也很简单，很多博客的写作用的就是markdown语法，比如简书，自己可以随便添加点什么，在hexo d，然后你可以访问一下你的博客了，比如我的https://hexiuhui.github.io/

配置文件，主题风格

接下来就是了解配置文件，修改成自己的风格的部分。

首先blog/_config.yml，看看注释基本上就了解啥意思了

主题选择

然后就是Dblog\themes\casual_config.yml，这个配置的是你选的主题，
会有不同。我选的主题是casual,
主题在这里可以随便选，选到喜欢的，就可以看看那个主题在github的介绍，一般作者会告诉你怎么用。
我用的这个主题，主题作者他给主题加入了评论，所以我们直接用就行。我也是今天刚用一下，感觉还可以，还有一些地方没修改过来，等空闲的时候在修改了，对了手机端看也挺漂亮的。

错误

1.ssh的时候出现了
ssh: connect to host github.com port 22: Connection timed out

2.hexo d的时候，一直报错，因为没仔细看，后来解决是因为没有执行git init

3.目前我用的是git@github.com的deploy，可是上传代码的时候还是要输入密码，这个应该是设置ssh的时候少了步骤，看这个教程跟着在设置一遍就好
